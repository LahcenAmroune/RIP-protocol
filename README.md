Le document présente RIP (Routing Information Protocol), un protocole de routage de couche 3 qui permet aux routeurs d’échanger leurs tables toutes les 30 secondes. Il utilise le nombre de sauts comme métrique, avec une limite de 15 sauts (16 = route inaccessible).

Il explique son fonctionnement, ses messages, ses temporisateurs, ainsi que ses limites (convergence lente, faible performance) et ses vulnérabilités.

Il y a aussi un fichier Cisco (Packet Tracer) avec un exemple pratique, notamment un cas de 16 sauts montrant une route inaccessible.
